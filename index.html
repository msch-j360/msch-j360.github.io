Goal To think about sizes other than desktop. At this point, we expect websites to work just as well on phones and tablets as they do on desktop.

Not just desktop anymore.

We're designing for all of these devices at the same time. Desktop, tablet, mobile ... watch? Not yet, but soon. All of these different platforms have different considerations. We have much less real estate on a tablet or a phone, so that means we have to design things different.

We'll start with a simple example and work our way up to something a little more complicated.

Secret: You've already been using them.

If you look at your news-site example from last week, and resize your browser, you should see all of the columns shrink down and eventually collapse into one. If you look inside the guts of your skeleton.css file, you'll notice that some of the css lines start with @media. Those are media queries. They allow us to target certain screen sizes and set style rules within them.

I added a media query to this page that changes the background if the page size is between 600 and 650px wide. Resize this browser window and watch the background color change.

Now inspect the css on this page to try to find the line that makes it do that. Can you modify the line to change the background color to cyan?

Let's start a new project.

Create a new git-repo inside your J360 folder called responsive-site.

Set up the project like normal. This should all feel routine. Just going through the motions. Create an index.html file, paste in the boilerplate HTML, create a css folder with a style.css file and a reset.css file. Link those files up in the head of your html file. Make sure everything is linked up and working before moving on.

The basic media query

In your index.html file, add two divs with classes and text like this.

In your style.css file, style the divs with a class of box to have a width and a height and a background color so we can see them. Know that you don't have to set widths in pixels. You can use percentages if you want a div to fill up a certain percentage of your browser.

Now, what we want to do with media queries is show the div with a class of show-desktop on desktop and the div with a class of show-mobile on mobile. A good breakpoint to use is 600 pixels. Show the mobile div below 600px and show the desktop div above 600px.

This is what that looks like. We hide the show-mobile class in your regular css, then in the media query, we show it by changing display: none; to display: block;. Same thing, except the opposite for the desktop version. We can use this to restyle things based on different viewport sizes. You can also add the show-desktop or show-mobile classes to any element you want to show or hide on desktop or mobile.

Your assignment this week is to keep exploring media queries and styling things for tablet and mobile. There are a few regular desktop to mobile patterns that I'm sure you're familiar with: 2 items side by side on desktop are stacked on mobile, navigation that is in line long line on desktop is collapsed into a hamburger menu on mobile, tables are collapsed down into only their essentials.

Try coding a few of these up. 2 images side by side on desktop, stacked on mobile. An alternate navigation bar on mobile/desktop. Slightly smaller text on mobile, to fit more text per screen. Images that stretch to 100% on mobile.
