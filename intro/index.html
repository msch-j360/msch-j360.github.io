OK, so what's this all about?

Brian Boyer, the visuals director at NPR, recently went on this mini-twitter rant.

I’m pissed at J-schools for creating young designers with awesome print layout skills.

— Brian Boyer (@brianboyer) August 18, 2015

Let’s try to change that.

This is going to be a more technical course than you’re probably used to.

A lot of people think a(n) Adobe product is going to come along that will make learning to code (and design, frankly) obsolete. That hasn’t happened yet. These days, web design is design and code. So yes, you’ll be learning to code. Black screen, white text, matrix-level code.

In this course we’ll be using tools that real people use in the real world. Nothing against OnCourse, Onestart, Canvas or anything else IU cooks up, but people don’t use those in real life. We’ll be using good ol’ email, Sublime Text, Github, Google Hangouts, Slack, Google Chrome and Adobe Illustrator.

Here’s how I imagine, in a dream world, things will go down.

I’ll upload and email one video lesson every Monday.

Length will vary, but I imagine it will be around an hour. (Considering that if this were an in-person class, we’d be meeting twice a week, 2 hours each, this seems like a steal.)

You’ll have one assignment per week, due on Friday before midnight.

You’ll get one of three grades. Done well (like an A), done (like a C), not done (like an F). Your final grade is all of your assignments averaged. No late assignments will be accepted. After an assignment is graded, you can resubmit as many times as you’d like for a better grade. Ambition will be rewarded over ease. You’re learning. This is hard and confusing. Failure while aiming high is fine. Not aiming high isn’t.

Throughout the week, I’ll hold office hours at various times for several hours.

Once I talk to all of you, I’ll try to figure out times that work for everyone. To help you when you get stuck and when your code won’t run, we’ll do this through a combination of Google Hangouts and Slack (group chat).

I want you to get a lot out of this course.

In this course I’m trying to give you skills that you can use immediately, add to your resume, and will help you get a job.

Specifically, at the end of this course you should

  • have better idea of how the internet works
  • know how the command line and git/github works
  • know how to design and code responsive websites with HTML/CSS and a (tiny) bit of javascript
  • understand the design principles that the web is built on
  • have several solid portfolio pieces (including a site to host them on)

To that end, I’ll get up early, stay up late, and otherwise help you as much as I can so you succeed. If something’s not working, please let me know. We’ll learn together, make it better and move on. If there’s something I’m not covering that you want me to, let me know. The syllabus can change.
